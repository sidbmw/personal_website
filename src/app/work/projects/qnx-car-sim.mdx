Contributed to the development of a QNX 7.1 based car simulator for COMP 4900, designed with a microkernel approach where each car function (steering, throttle, brakes, indicators, airbags, engine state) operates as a separate process.

Implemented a client-server architecture using Inter-Process Communication (IPC) for real-time message handling, with the engine process acting as the server.

**Specific Contributions:**
*   Set up QNX development environments for team members.
*   Configured virtual machines using `mkqnximage`.
*   Explored various front-end/UI possibilities for QNX (Python GUI libraries like PySimpleGUI, tkinter, Remi; HTML/PHP/JS).
*   Authored `simulator.sh` and `demo.sh` scripts for running simulations.
*   Reviewed pull requests, suggested changes, compiled, and tested all code within a QNX virtual machine environment.
